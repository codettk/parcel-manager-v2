# 필지 리브랜딩 + 전국 지적도 지역 선택 진입

- 상태: 검토 대기
- 매핑: 신규 (M-1~M-18 매핑표 밖 — 제품 방향 전환 슬라이스. 근거: `memory/pivot-pilji-national.md`)
- 판정:
  - **리브랜딩**: 재설계 (보구곶 단일 마을 텍스트·브랜드색을 범용 "필지" 정체성으로 교체. v1/v2 기존 카피·녹색 토큰 폐기)
  - **지역(region) 추상화**: 신규 (v2에 region 개념 부재 — 단일 작업공간만 존재)
  - **지도 코어(렌더·제스처·시트·동기화)**: 보존 (M-2~M-16 산출물은 region 1개 한정으로 그대로 동작 — 이번 슬라이스는 그 위에 진입 게이트만 얹는다)

## 배경 / 절충 (반드시 후속 단계가 인지)

보구곶리 단일 마을 앱을 전국 지적도 범용 토지 도구 "필지"의 첫 슬라이스로 전환한다.
**전국 지적도 데이터 파이프라인은 이번 범위 밖이다.** 따라서:

- 보구곶리(= 인천 강화군 화도면)를 **유일하게 데이터가 적재된 region** 으로 시드한다.
- region 추상화(목록·현재 region·전환 진입점)는 도입하되, 실제 지적도(`public/data/parcels.json` 단 하나)는 보구곶 region에만 연결한다.
- 검색 결과·지역 관리 목록에 보구곶 외 행정구역이 노출될 수 있으나, **데이터 미적재 region 은 "준비 중"(비활성) 으로만 표시**하고 선택해도 지도로 진입하지 않는다.
- 이 절충은 region 모델·API 가 1개→N개 region 으로 자연 확장 가능한 형태(목록/현재/전환)를 갖추되, 지도 로딩만 단일 데이터셋에 고정함으로써 후속 데이터 파이프라인 슬라이스를 비파괴적으로 받아들이는 것이 목적이다.

## 사용자 스토리

1. 처음 앱을 연 사용자로서, 보구곶 영농 전용이 아니라 "필지 — 전국 지적도" 라는 범용 서비스임을 화면 카피·브랜드로 인지하고 싶다 (잘못된 대상 인식 방지).
2. 새 사용자로서, GPS 또는 행정구역 검색으로 내가 볼 지역을 고른 뒤 지도로 진입하고 싶다 (전국 어디서든 시작점이 명확해야 한다).
3. 기존 사용자로서, 지도 상단에서 현재 보고 있는 지역을 항상 확인하고, 칩을 눌러 다른 지역으로 전환하거나 지역을 관리(추가·제거 — 이번 슬라이스는 열람만)하고 싶다.

## 수용 기준 (AC)

### 리브랜딩

AC-1. (Given 앱이 부팅을 마친 상태) When 사용자가 앱 메뉴 드로어를 연다 Then 화면 어디에도 "보구곶리" "영농" 문구가 노출되지 않고, 앱 브랜드명으로 "필지"가 표시된다.

AC-2. (Given 브라우저 탭) When 앱 문서가 로드된다 Then 문서 `<title>`이 "필지"를 포함하고 "보구곶리"를 포함하지 않는다.

AC-3. (Given 토큰 기반 스타일) When 브랜드 강조 요소(주요 CTA·선택 강조)가 렌더된다 Then `--color-primary` 토큰이 브랜드 블루(#2C5FD0)로 적용되어, 해당 요소의 계산된 색이 기존 녹색(#2f7d4f)이 아니다.

### 지역 선택 진입 게이트

AC-4. (Given localStorage 에 마지막 선택 region 기록이 없는 첫 방문) When 앱이 부팅을 마친다 Then 지도 대신 "지역 선택" 화면이 표시되고, 지도 캔버스·탭바는 보이지 않는다.

AC-5. (Given 지역 선택 화면) When 사용자가 검색창에 "화도" 를 입력한다 Then 결과 목록에 "인천 강화군 화도면(보구곶)" 항목이 데이터 적재됨(활성)으로 표시되고, 이를 탭하면 지도 화면으로 전환되며 보구곶 필지(parcels.json)가 렌더된다.

AC-6. (Given 지역 선택 화면) When 사용자가 데이터 미적재 행정구역(예: 임의 검색 결과 중 비활성 항목)을 탭한다 Then "준비 중" 안내가 표시되고 지도로 전환되지 않는다 (지도 캔버스 미표시 유지).

AC-7. (Given GPS 진입 동선) When 사용자가 "현재 위치로 찾기"를 탭하고 위치 권한이 거부/미지원이다 Then 디자인된 권한 안내가 표시되고 앱이 중단되지 않으며, 검색으로 지역을 고르는 경로가 계속 사용 가능하다. (실제 역지오코딩 백엔드는 비범위 — 권한 분기·폴백 UI만 검증)

### 지역 전환 / 지도 지역칩

AC-8. (Given 보구곶 region 으로 지도에 진입한 상태) When 지도 화면이 렌더된다 Then 지도 상단에 현재 region 명("화도면(보구곶)")을 표시하는 지역칩이 보인다.

AC-9. (Given 지도 화면의 지역칩) When 사용자가 지역칩을 탭한다 Then 지역 선택(또는 지역 관리) 화면이 다시 열려 다른 지역으로 전환하거나 현재 지역을 바꿀 수 있는 진입점이 제공된다.

AC-10. (Given 보구곶 region 을 한 번 선택해 localStorage 에 기록된 상태) When 사용자가 앱을 새로고침한다 Then 지역 선택 화면을 거치지 않고 곧바로 보구곶 지도로 복귀한다 (마지막 선택 region 영속).

AC-11. (Given 앱 메뉴 드로어) When 사용자가 "지역 관리" 항목을 탭한다 Then 받은(데이터 적재) 지역 목록이 표시되고, 보구곶 region 이 목록에 존재하며, 각 항목에서 해당 지역으로 전환할 수 있다.

## 비범위

- 전국 지적도 데이터 적재·다중 region parcels 파이프라인 (보구곶 1개 region 만 데이터 보유).
- 실제 GPS 역지오코딩 백엔드 (권한 분기·폴백 UI 만 — 좌표→행정구역 변환 없음).
- 네이티브 소셜 로그인·토큰 핸드오프 실제 구현 (디자인만 존재 — 웹앱 범위 밖). 로그인 게이트는 이번 슬라이스에서 강제하지 않는다.
- PRO(영농 ERP) 구독·freemium 게이팅·페이월·IAP.
- 지역 "추가/다운로드/제거" 실제 동작 (관리 화면은 적재 region 열람·전환만 — 데이터 받기 액션은 후속).
- 기존 작업공간(tabs)·필지 편집·그룹·팔레트·계산기 동작 변경 (region 1개 한정으로 그대로 보존).

## 영향 범위

- 프론트:
  - 신규 `src/features/region/` — `RegionSelectView`(GPS priming + 검색 + 최근/받은 목록), `RegionManageView`(받은 지역 열람·전환), `RegionChip`(지도 상단 현재 region 표시·전환 진입점), `regionCatalog.ts`(시드 region 메타 + "준비 중" 분류 순수 로직).
  - `src/stores/ui.ts` — region 진입 게이트 상태(현재 region·선택 화면 열림·localStorage 영속 키 `pilji_v2_active_region`). 부팅 시 마지막 region 미기록이면 게이트 표시.
  - `src/App.tsx` — region 미선택 시 지도/탭바 대신 `RegionSelectView` 렌더 분기, 지도 상단 `RegionChip` 배치(TabBar 행 또는 지도 오버레이), `NavDrawer` 에 "지역 관리" 항목 추가.
  - `src/features/tab/NavDrawer.tsx` — "지역 관리" DrawerItem 추가.
  - 리브랜딩: `index.html`(title), `src/styles/tokens.css`(`--color-primary` 등 브랜드색 블루 전환 — 녹색→블루는 토큰 단일 변경), 화면 카피 전수 교체. `package.json` name 은 선택(후속).
- 백엔드: **없음** (region 추상화는 클라이언트 메타·localStorage 로 구현. 단일 데이터셋·단일 작업공간 구조 무변경. 후속 다중 region 시 API 도입).
- DB: **마이그레이션 불필요** (region 은 DB 개념으로 승격하지 않는다 — 절충에 따라 보구곶 단일 데이터셋 고정. tabs/parcels 스키마 무변경).
- API 계약: **없음** (신규/변경 엔드포인트 없음. region 메타·검색 카탈로그는 `src/features/region/regionCatalog.ts` 클라이언트 상수 — zod 계약 대상 아님. 향후 다중 region 데이터 적재 시 `src/types/api/` 에 region 스키마 신설 예정).
