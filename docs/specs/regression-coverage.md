# 회귀 커버리지 대조표 (Phase 5 §8.3)

- 작성일: 2026-06-16
- 기준: 마이그레이션 명세서 §8.2 회귀 항목 — "v1 `docs/tests/` 시나리오(reset-history 등)를 v2 구조로 포팅"
- 원칙: **1:1 복사 금지.** v2는 v1과 의도적으로 다르다(§7.3 — 스냅샷/히스토리 UI 폐기 → 탭/히스토리). v1이 검증한 **행동 중 v2가 아직 커버 안 한 회귀 갭만** 포팅한다. 억지 중복 테스트 금지.

| ----------------------- | ----- | ----------------------------------------------------------------------------------------------------------- |

| **폐기(재설계로 대체)** | 17    | v2가 스냅샷/히스토리 UI를 탭으로 대체 — 검증 대상 자체가 v2에 존재하지 않음. §7.3·§8.1로 확정된 의도된 변경 |

## D. 갭 판정 — 포팅 0건 근거

검토한 모든 v1 검증 행동에 대해:

1. **스냅샷/히스토리 행동(기능 2~6 전부, 1-5)** = §7.3-1로 확정된 의도된 폐기. v2는 스냅샷 테이블·API·인라인 편집 UI가 **존재하지 않으므로** 포팅 대상 자체가 없다. 등가 흐름(탭 생성→전환→소프트클로즈→복원)은 `tab-workspace.spec.ts`·`journey-3-tab-lifecycle.spec.ts`가 이미 green으로 커버.
2. **초기화 핵심 행동(1-1~1-4, partial-preserve)** = v2 `reset.spec.ts`(E2E 픽셀) + `workspaceReset.test.ts`(액션 단위, 부분 보존 명시) + `override.test.ts`(patch) + 통합 `tabState.test.ts` AC-10(pinned 보호·부수효과 0)으로 다층 커버.
3. **면적/계산기 행동(전 항목)** = E2E `calculator.spec.ts`·`parcel-list.spec.ts`·`parcel-sheet.spec.ts` + RTL `CalculatorResultSheet`/`CalculatorSettingsSheet`/`ParcelSheet` + `formatArea`/`calc` 단위로 전부 커버.

v1이 검증했으나 v2 어디에도 등가물이 없는 회귀 행동은 발견되지 않았다. 신규 E2E 작성 시 위 기존 spec과 행동이 겹쳐 억지 중복이 되므로, **포팅 0건 + 본 대조표로 "전부 커버됨" 입증**이 정당한 산출물이다(작업 지시 3항).

### 참고: v2 재설계가 만든 신규 회귀 자산 (v1에 없던 보강)

- **탭 격리**: `tab-workspace.spec.ts` AC-1 (읽기·쓰기 양방향 격리 + round-trip 복원) — v1 스냅샷엔 없던 행동.
- **pinned × 색만 초기화 격리**: `reset.spec.ts` AC-11 — 색 항목만 비우고 pinned 색 보존을 픽셀로 검증(v1은 전체 초기화 + 개수 불변만).
- **계산기 그룹/개별 재계산**: `calculator.spec.ts` AC-12 (그룹 합산 164.6 ↔ 개별 82.3 토글) — v1엔 그룹 합산 결과 토글 없음.
