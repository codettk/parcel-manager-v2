# 릴리즈 노트 시트

- 상태: 출시 준비 (2026-06-11)
- 매핑: M-17 (파이프라인 파일럿 런 대상)
- 판정: 재설계 (시트 UI·정보 구조는 v1 보존하되, 런타임 fetch + `dangerouslySetInnerHTML` 파싱을 빌드 타임 구조화 데이터로 교체 — 네트워크 실패 상태와 innerHTML 주입 표면 제거)

## 사용자 스토리

1. 공동체 사용자는 앱이 어떻게 바뀌었는지 확인하기 위해, 지도 화면에서 릴리즈 노트를 열어 버전별 변경사항 목록을 시트로 본다.
2. 운영자(배포자 에이전트 포함)는 배포 시 릴리즈 노트 파일(`RELEASE_NOTES.md`) 한 곳만 갱신하면 별도 코드 수정 없이 사용자 화면에 반영되기를 원한다.

## 동작 명세 (v1 기준 + 재설계 차이)

- **데이터 소스**: 저장소 루트 `RELEASE_NOTES.md` (v2 자체 노트, v1 내용 이관 없음). Vite `?raw` import 등 빌드 타임 포함 — 런타임 fetch 없음. 따라서 v1의 "불러오는 중…" / "불러올 수 없습니다" 상태는 v2에 존재하지 않는다.
- **파싱 규칙** (v1 `parseReleaseNotes` 보존): `## ` = 버전 섹션, `### ` = 그룹 헤딩(새로운 기능/개선사항/버그 수정), `- ` = 항목, 항목 내 `**텍스트**` = 강조. 그 외 마크다운 문법은 미지원.
- **표시**: 시트 헤더(메타 "앱 정보" / 제목 "릴리즈 노트"), 버전 섹션 카드가 파일 기재 순서(최신이 위)대로 나열, 각 카드에 버전 문자열 + 그룹 헤딩 + 불릿 항목. 강조는 `<strong>` 요소로 렌더 (innerHTML 금지 — 파서가 구조화 토큰으로 분해).
- **진입점(최소)**: v2에 NavDrawer가 아직 없으므로, 지도 화면 우상단에 임시 `IconButton`(lucide `scroll-text`) 1개를 두어 시트를 연다. NavDrawer 도입 시 이 버튼은 드로어 항목("릴리즈 노트 — 버전별 업데이트 내역")으로 이동한다.
- **시트 컨테이너**: 공통 `Sheet` 사용 (뷰포트에 따라 BottomSheet/SidePanel 자동 — 400ms 닫힘 가드 포함, 컨테이너 책임이므로 본 기능 AC 아님).

## 수용 기준 (AC)

AC-1. Given 지도 화면, When 우상단 릴리즈 노트 버튼(접근성 이름 "릴리즈 노트")을 탭하면, Then 릴리즈 노트 시트가 열리고 제목 "릴리즈 노트"가 표시된다.

AC-2. Given 릴리즈 노트 시트가 열린 상태, Then `RELEASE_NOTES.md`의 모든 `## ` 버전 섹션이 파일 기재 순서대로 표시되고, 각 섹션에 버전 문자열·`### ` 그룹 헤딩·`- ` 불릿 항목 텍스트가 모두 렌더된다.

AC-3. Given 항목 텍스트에 `**강조**` 마크다운이 포함된 경우, Then 화면에 `**` 마커가 노출되지 않고 해당 텍스트가 `<strong>` 요소로 렌더된다.

AC-4. Given 릴리즈 노트 시트가 열린 상태, When 닫기 버튼을 탭하면, Then 시트가 닫히고 지도 화면으로 복귀하며 릴리즈 노트 버튼이 다시 조작 가능하다.

AC-5. Given 빈 그룹·헤딩 없는 불릿 등 파싱 불가 라인이 섞인 입력, When 파서(`parseReleaseNotes`)에 넣으면, Then 규칙에 맞는 라인만 구조화되고 예외 없이 결과를 반환한다 (단위 테스트로 검증).

## 비범위

- **NavDrawer / 메뉴 시스템 전체** — `Drawer`·`DrawerItem` 컴포넌트는 존재하나 드로어 화면 자체는 별도 기능. 본 건은 임시 진입 버튼 1개까지만.
- TopBar 등 지도 화면 상단 UI 체계화.
- v1 `RELEASE_NOTES.md` 내용 이관 — v2는 별도 서비스이므로 v2 자체 첫 항목 1개(예: "버전 2026-06-XX — v2 파일럿")만 시드.
- 일반 마크다운 렌더러 (링크·이미지·코드 등 미지원, 강조만).
- 릴리즈 노트 자동 생성/배포 연동 (deployer 단계 소관, 추후).
